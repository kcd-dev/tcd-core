Phase 0 · 失败起点（2023）
KCD 1.0
知识收敛（Knowledge Convergence Development）
写文档 → 失败
▶

「让 AI 记住规则并执行 → 失败的开端」

文档驱动知识积累静态规则
  • 核心思路：写详细的 CLAUDE.md 文档，让 AI 记住规则
  • 失败原因：AI 第一次执行完美，第二次就忘记规则
  • 写了详细文档，AI 还是会丢步骤
  • 换个 session，AI 要重新问「你想做什么」
文档失效 规则遗忘 Session断连
Phase 1 · 范式革命（2024 Q1–Q3）
TCD 2.0
范式转换：AI 翻译 + 脚本执行
核心突破 · 一次翻译，N次执行
▶

「AI 是翻译器，不是执行者——让 AI 生成脚本，程序执行任务」

AI擅长: 需求→代码（翻译，概率性） 程序擅长: 代码→执行（确定性） ═══════════════════════════════ 一次翻译 → N次执行（成本最优）
  • 收敛 = 固化为脚本，不是写文档
  • AI 角色：翻译器（需求 → 结构化代码）
  • 程序角色：执行器（确定性、可重复）
错误率 -100% 成本 -56% 效率 +76%
TCD 3.0
三层架构
人类看TXT / AI看JSON / 自动化用脚本
▶

「分层才能分治——三类受众，三种格式」

人类可读层（TXT）: claude-progress.txt, git-log.sh AI 可读层（JSON）: feature_list.json, workflows.json 脚本层（自动化）: init.sh, start.sh, stop.sh, healthcheck.sh
  • 改需求 → 优先改 config 或 workflows，少动 core
  • init.sh：环境预检（30秒），每个 session 必跑
  • healthcheck.sh：API 健康检查，执行前验证服务
TCD 4.5
AI 友好技术栈
TypeScript 优先，强类型即文档
▶

「技术栈对 AI 的可理解性差异巨大——选择对 AI 友好的技术」

优先级: TypeScript > Python > Bash Web: Next.js + TypeScript（约定优于配置） CLI: TypeScript(tsx) 或 Python 后台: Rust（性能）/ Node.js（生态）
  • 强类型 = AI 更少猜测，类型即文档
  • pnpm 而不是 npm，Vue 而不是纯 HTML
  • 前后端分离，分开文件夹存放
Phase 2 · 工程化深化（2024 Q4 – 2025 Q2）

▶

「每个脚本和模块必须包含自我验证逻辑」

Layer 1: Pre-Validation → 前置检查（依赖/参数/环境） Layer 2: Execution → 执行中验证（关键步骤后立即验证） Layer 3: Post-Validation→ 后置验证（最终状态确认） Layer 4: Report → 验证报告（N/N 通过 + 失败原因）

核心演进脉络

从失败到范式（KCD 1.0 → TCD 3.0）

KCD 1.0 的根本错误：让 AI 记住规则执行。TCD 2.0 的范式革命：AI 做翻译（概率性），程序做执行（确定性）。「一次翻译，N次执行」成为整个 TCD 体系的基石。

工程化深化（TCD 8.0 → 15.10）

验证工程化 → 资源效率 → 以人为本 → 主动进攻 → 多AI共识 → 数据壁垒 → 自验证 → 情绪增强 → Clawn（数字劳动力）。每个版本解决上一版本遗留的一个核心问题。

情报驱动（TCD 16.0 → 17.0）

从「系统自愈」到「情报先行」。TCD 17.0 提炼 Palantir 战场系统精髓：结构化情报是 AI 决策的天花板，而非模型能力。决策压缩才是真正的商业价值。

记忆革命（TCD 18.0 → 18.1）

TCD 18.0 用三层架构 + Claude Code Hooks 实现零维护知识系统，节省 68% 上下文。TCD 18.1 引入时间约束和赛马机制，AI 与人类双赛道竞争，合并后质量提升 5%+。

生态壁垒（技能市场 + CLI市场）

96个 Skills + 13+个定制CLI 构成了不可复制的工具壁垒。TCD 15.2 的「数据壁垒」在工具层的具体体现：别人可以抄袭理念，但无法复制这套经过真实场景验证的工具生态。

终局推演（TCD 20.0）

「技术奇点不是人类的终点，而是角色重新定义的起点。」TCD 20.0 的核心洞察：在 AGI 时代，工程师的终极价值不是写代码，而是定义代码应当服务的人类价值——这是任何AI无法替代的。
